# Export System

This guide describes the export system in LiveCodes, located in `src/livecodes/export/`.

## Overview

The export system allows exporting projects to various formats including JSON, ZIP (source), HTML (result), GitHub Gist, CodePen, and JSFiddle.

## Export Types

| Type | Handler | Output | Description |
|------|---------|--------|-------------|
| `json` | `exportJSON` | `.json` file | Project configuration |
| `src` | `exportSrc` | `.zip` file | Source files + result + config |
| `html` | `exportHTML` | `.html` file | Standalone result page |
| `codepen` | `exportCodepen` | CodePen URL | Prefilled CodePen pen |
| `jsfiddle` | `exportJsfiddle` | JSFiddle URL | Prefilled JSFiddle |
| `githubGist` | `exportGithubGist` | Gist URL | Public GitHub gist |

## Utility Functions (utils.ts)

### getFilesFromConfig

Extracts file structure from configuration:

```mermaid
flowchart TD
    A[Config] --> B{Has content?}

    B -->|markup| C["index.{ext}"]
    B -->|style| D["style.{ext}"]
    B -->|script| E["script.{ext}"]
    B -->|stylesheets| F[styles file<br/>external links]
    B -->|scripts| G[scripts file<br/>external links]
    B -->|tests| H["script.spec.{ext}"]

    C --> I[Files object]
    D --> I
    E --> I
    F --> I
    G --> I
    H --> I
```

### getContent

Handles language support for export targets:

Supported languages vary by platform:
- **CodePen**: markdown, haml, less, scss, sass, stylus, babel, typescript, coffeescript, livescript
- **JSFiddle**: haml, scss, sass, babel, typescript, coffeescript

For unsupported languages, compiled code is used.

### getCompilerScripts

External compiler scripts needed for non-JS languages:

```mermaid
flowchart LR
    A[Language] --> B{Supported?}
    B -->|Yes| C[Use original code]
    B -->|No| D[Get compiler scripts]
    D --> E[Filter HTTPS URLs]
    E --> F[Add to external resources]
```

## Export Handlers

### JSON Export (export-json.ts)

Exports project configuration:

```mermaid
sequenceDiagram
    participant Core
    participant ExportJSON
    participant Browser

    Core->>ExportJSON: exportJSON(config)
    ExportJSON->>ExportJSON: Stringify config
    ExportJSON->>Browser: Download as {title}.json
```

### Source Export (export-src.ts)

Creates ZIP with all project files:

```mermaid
sequenceDiagram
    participant Core
    participant ExportSrc
    participant JSZip
    participant Browser

    Core->>ExportSrc: exportSrc(config, html)
    ExportSrc->>JSZip: Load JSZip
    ExportSrc->>ExportSrc: getFilesFromConfig()

    loop Each file
        ExportSrc->>JSZip: zip.file(filename, content)
    end

    ExportSrc->>JSZip: zip.file(result.html)
    ExportSrc->>JSZip: zip.file(livecodes.json)
    JSZip->>ExportSrc: base64 output
    ExportSrc->>Browser: Download as {title}.zip
```

### HTML Export (export-html.ts)

Exports standalone result page:

```mermaid
sequenceDiagram
    participant Core
    participant ExportHTML
    participant ResultPage
    participant Browser

    Core->>ResultPage: getResultPage(forExport=true)
    ResultPage-->>Core: HTML string
    Core->>ExportHTML: exportHTML(config, html)
    ExportHTML->>Browser: Download as {title}.html
```

### CodePen Export (export-codepen.ts)

Creates prefilled CodePen:

```mermaid
flowchart TD
    A[exportCodepen] --> B[Check language support]
    B --> C{Supported?}
    C -->|Yes| D[Use original code]
    C -->|No| E[Use compiled code]

    D --> F[Build form data]
    E --> F

    F --> G[Add preprocessor settings]
    G --> H[Add external resources]
    H --> I[POST to codepen.io/pen/define]
    I --> J[Open in new window]
```

### JSFiddle Export (export-jsfiddle.ts)

Creates prefilled JSFiddle:

```mermaid
flowchart TD
    A[exportJsfiddle] --> B[Check language support]
    B --> C[Build form data]
    C --> D[Set panel types<br/>CSS:1, JS:4 for TypeScript]
    D --> E[Add external resources]
    E --> F[POST to jsfiddle.net/api/post/library/pure/]
    F --> G[Open in new window]
```

### GitHub Gist Export (export-github-gist.ts)

Creates public GitHub gist:

```mermaid
sequenceDiagram
    participant Core
    participant ExportGist
    participant GitHub API

    Core->>ExportGist: exportGithubGist(config, user)
    ExportGist->>ExportGist: getFiles(config)
    ExportGist->>GitHub API: POST /gists

    alt Success
        GitHub API-->>ExportGist: gist.id, gist.html_url
        ExportGist->>ExportGist: Update gist with project link
        ExportGist->>GitHub API: PATCH /gists/{id}
        GitHub API-->>ExportGist: Updated gist
        ExportGist->>Core: Open gist URL
    else Failure
        GitHub API-->>ExportGist: Error
        ExportGist-->>Core: No action
    end
```

## Usage in core.ts

### handleExport Function

Main entry point for export functionality:

```mermaid
flowchart TD
    A[handleExport] --> B[Load export module<br/>lazy loaded]

    B --> C[Export JSON<br/>click handler]
    B --> D[Export Result<br/>click handler]
    B --> E[Export Source<br/>click handler]
    B --> F[Export CodePen<br/>click handler]
    B --> G[Export JSFiddle<br/>click handler]
    B --> H[Export GitHub Gist<br/>click handler]

    C --> I[updateConfig]
    I --> J[exportJSON]

    D --> K[updateConfig]
    K --> L[getResultPage forExport=true]
    L --> M[exportConfig html]

    E --> N[updateConfig]
    N --> L
    N --> M2[exportConfig src]

    F --> O[updateConfig]
    O --> P{Cache valid?}
    P -->|No| Q[getResultPage]
    P -->|Yes| R[getCachedCode]
    Q --> R
    R --> S[exportConfig codepen<br/>with compiled]

    G --> T[Similar to CodePen]
    T --> U[exportConfig jsfiddle]

    H --> V[Get authenticated user]
    V --> W{User logged in?}
    W -->|No| X[Return]
    W -->|Yes| Y[exportConfig gist]
```

### Lazy Loading

Export handlers are lazy-loaded to reduce initial bundle size:

```typescript
let exportModule: typeof import('./export/export');
const loadModule = async () => {
  exportModule = exportModule || (await import(baseUrl + '{{hash:export.js}}'));
};
```
