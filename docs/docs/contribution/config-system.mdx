# Configuration System

This document describes the configuration system which manages application state in LiveCodes. It outlines the architecture, data flow, and how configurations are managed.

## Overview

LiveCodes configuration system handles:

- Default configuration values
- User configuration merging
- URL query parameter parsing
- Configuration validation
- Configuration version upgrades

For individual configuration properties and their meanings, see the [Configuration Documentation](https://livecodes.io/docs/configuration).

## Architecture

```
src/livecodes/config/
├── index.ts           # Public API exports
├── default-config.ts  # Default configuration values
├── build-config.ts    # Configuration builder (merges defaults + user + params)
├── config.ts          # Config getters/setters and helpers
├── validate-config.ts # Configuration validation
└── upgrade-config.ts  # Version upgrade migrations
```

## Configuration Flow

### 1. Default Configuration

`default-config.ts` provides the base configuration:

```typescript
export const defaultConfig: Config = {
  title: 'Untitled Project',
  description: '',
  head: `<meta charset="UTF-8" />\n<meta name="viewport" content="width=device-width, initial-scale=1.0" />`,
  autoupdate: true,
  autosave: false,
  delay: 1500,
  theme: 'dark',
  markup: { language: 'html', content: '' },
  style: { language: 'css', content: '' },
  script: { language: 'javascript', content: '' },
  // ... more defaults
  version: process.env.VERSION as string,
};
```

### 2. Build Configuration

`build-config.ts` merges configuration sources in order of precedence:

```typescript
export const buildConfig = (appConfig: Partial<Config>): Config => {
  // 1. Start with default config
  // 2. Merge user-provided config
  // 3. Override with URL query parameters
  // 4. Fix language names (aliases to canonical names)const userConfig = upgradeAndValidate(appConfig);

  let config: Config = {
    ...defaultConfig,
    ...userConfig,
  };

  // Get URL params and merge
  const params = getParams();
  const { version, ...paramsConfig } = upgradeAndValidate(loadParamConfig(config, params));

  config = { ...config, ...paramsConfig };

  // Fix language aliases
  return fixLanguageNames(config);
};
```

### 3. Configuration State

`config.ts` manages runtime configuration state with immutable getters/setters:

```typescript
let appConfig = defaultConfig;

// Get current configuration
export const getConfig = (): Config => cloneObject(appConfig);

// Set new configuration
export const setConfig = (newConfig: Config) => {
  appConfig = cloneObject(newConfig);
};

// Extract content-only config (for sharing/export)
export const getContentConfig = (config: Config): ContentConfig => {
  /* ...*/
};

// Extract app-only config
export const getAppConfig = (config: Config): AppConfig => {
  /* ... */
};

// Extract user preferences
export const getUserConfig = (config: Config): UserConfig => {
  /* ... */
};

// Upgrade and validate in one step
export const upgradeAndValidate = (config: Partial<Config>) =>
  validateConfig(upgradeConfig(config as any));
```

## URL Parameter Handling

### Parameter Types

The system supports multiple [URL query parameter](https://livecodes.io/docs/configuration/query-params) formats (with the type `UrlQueryParams`)

```
?js                                    # Language shorthand
?lang=js                               # Set active language
?html=hi&scss&ts                       # Set editor content + language
?languages=html,md,css,ts              # Enable multiple languages
?processors=tailwindcss,autoprefixer # Enable processors
?console                              # Open console (shorthand)
?console=open                         # Open console (explicit)
?tests=full                           # Open tests in full mode
?lite                                 # Enable lite mode
?customSettings={...}                 # Custom settings as JSON
?markup.hideTitle=true                # Nested properties
```

### Load Param Config

`loadParamConfig` converts URL parameters to configuration:

```typescript
export const loadParamConfig = (config: Config, params: UrlQueryParams): Partial<Config> => {
  const paramsConfig: Partial<Config> = {};

  // ?html=hi&scss&ts - Set editor content/language
  Object.keys(params).forEach((key) => {
    const language = getLanguageByAlias(key);
    const editorId = getLanguageEditorId(language);
    if (editorId) {
      paramsConfig[editorId] = { language, content: params[key] };
    }
  });
  // ?activeEditor=style or ?active=1
  // ?languages=html,md,css,ts
  // ?processors=tailwindcss,autoprefixer
  // ?tools=none|open|full
  // ?customSettings={...}
  // ?markup.hideTitle=true (nested props)

  return paramsConfig;
};
```

## Configuration Validation

`validate-config.ts` ensures configuration validation in runtime:

```typescript
export const validateConfig = (config: Partial<Config>): Partial<Config> => {
  // Type checking helpers
  const is = (x: any, type: types, arrayType?: types): boolean => {
    /* ... */
  };

  // Validate each property
  return {
    ...(is(config.title, 'string') ? { title: config.title } : {}),
    ...(is(config.autoupdate, 'boolean') ? { autoupdate: config.autoupdate } : {}),
    ...(includes(views, config.view) ? { view: config.view } : {}),
    // ... validate all properties

    // Validate nested objects
    ...(is(config.markup, 'object')
      ? { markup: validateEditorProps(config.markup, 'markup') }
      : {}),
    // ...
  };
};
```

## Usage

### Getting/Setting Configuration

```typescript
// In core.ts
import { getConfig, setConfig, buildConfig } from './config';

// Get current config
const config = getConfig();

// Set new config
setConfig({ ...getConfig(), title: 'New Title' });

// Build config from user input
const newConfig = buildConfig(userInput);
```

### Configuration Categories

The system extracts specific configuration subsets:

```typescript
// Content config (for sharing/export)
const contentConfig = getContentConfig(getConfig());
// Returns: { title, description, markup, style, script, ... }

// App config (behavior settings)
const appConfig = getAppConfig(getConfig());
// Returns: { readonly, allowLangChange, view, mode, tools, zoom }

// User config (preferences)
const userConfig = getUserConfig(getConfig());
// Returns: { autoupdate, autosave, theme, editorTheme, ... }

// Editor config
const editorConfig = getEditorConfig(getConfig());
// Returns: { editor, theme, fontFamily, fontSize, ... }

// Formatter config
const formatterConfig = getFormatterConfig(getConfig());
// Returns: { useTabs, tabSize, semicolons, singleQuote, trailingComma }
```

## Configuration Sources Priority

When building the final configuration, sources are merged in this order (later overrides earlier):

1. **Default configuration** (`default-config.ts`)
2. **Saved project configuration** (from storage)
3. **User configuration** (programmatic - e.g. SDK)
4. **URL query parameters** (including compressed params)
5. **URL hash parameters** (highest priority)

Example:

```typescript
// Default: theme = 'dark'
// Saved user config: theme = 'light'
// URL param: ?theme=dark
// Final: theme = 'dark'
```

## Configuration Persistence

Configuration is persisted through the storage system:

```typescript
// Save project
const projectConfig = buildConfig(getConfig());
await stores.projects?.addItem(projectConfig);

// Load project
const savedConfig = await stores.projects?.getItem(projectId);
setConfig(buildConfig(savedConfig));
```

## Special Configuration Handling

### Language Aliases

Language names are normalized to canonical names:  
(e.g. `md` becomes `markdown`, `ts` becomes `typescript`)

```typescript
const fixLanguageNames = (config: Config): Config => ({
  ...config,
  markup: {
    ...config.markup,
    language: getLanguageByAlias(config.markup.language) || 'html',
  },
  // ... same for style, script
});
```

## Adding New Configuration Properties

1. **Add type definition** in `src/sdk/models.ts`:

   ```typescript
   export interface EditorConfig {
     // ...
     /** Add descriptive JSDoc */
     newProperty: string;
   }
   ```

2. **Add default value** in `src/livecodes/config/default-config.ts`:

   ```typescript
   export const defaultConfig: Config = {
     // ...
     newProperty: 'default value',
   };
   ```

3. **Add validation** in `src/livecodes/config/validate-config.ts`:

   ```typescript
   return {
     // ...
     ...(is(config.newProperty, 'string') ? { newProperty: config.newProperty } : {}),
   };
   ```

4. **Add upgrade step** if needed in `src/livecodes/config/upgrade-config.ts`.

5. **Add it do a configuration category** in `src/livecodes/config/config.ts`:

   ```typescript
   export const getEditorConfig = (config: Config): EditorConfig => ({
     // ...
     newProperty: config.newProperty,
   });
   ```

6. **Document the property** in `docs/docs/configuration/configuration-object.mdx`.
