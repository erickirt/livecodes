livecodes.io / icon redesign proposal

A sharper mark
for a sharper tool

The current logo is a wide illustration that doesn't work as an icon. This proposal replaces it with a single, focused symbol built on one idea: code brackets + a live indicator. Readable from 512 px down to 16 px.

current logo
Current design

Wide illustration with two gray brackets flanking an isometric cube. Muted grays, dated 3D style, complex nested geometry.

  • 737 × 488 — unusable as app icon or favicon
  • Dominant gray palette — no energy or identity
  • Isometric 3D style — circa 2015 aesthetic
  • Three competing symbols — brackets + 2 cubes
  • Collapses into noise at 32 px
proposal
Proposed design

Square dark icon. One focused symbol: code brackets + a live indicator. Works at every size, in every context.

  • 512 × 512 — scales to any context perfectly
  • Electric cyan on deep navy — distinct and modern
  • Flat with precision glow — timeless, not trendy
  • Single symbol — </> reads in under a second
  • Live dot survives at 16 px as a warm accent

<!-- LiveCodes Icon -->
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512">
  <defs>

    <!-- Background gradient -->
    <linearGradient id="bg" x1="0%" y1="0%" x2="100%" y2="100%">
      <stop offset="0%"   stop-color="#0d1825"/>
      <stop offset="100%" stop-color="#080d16"/>
    </linearGradient>

    <!-- Subtle dot texture -->
    <pattern id="dots" x="0" y="0" width="28" height="28" patternUnits="userSpaceOnUse">
      <circle cx="1" cy="1" r=".75" fill="rgba(255,255,255,.04)"/>
    </pattern>

    <!-- Cyan glow on the </> glyph -->
    <filter id="glow" x="-40%" y="-40%" width="180%" height="180%">
      <feGaussianBlur stdDeviation="14" result="b"/>
      <feFlood flood-color="#00c8ff" flood-opacity=".32" result="c"/>
      <feComposite in="c" in2="b" operator="in" result="cb"/>
      <feMerge>
        <feMergeNode in="cb"/>
        <feMergeNode in="SourceGraphic"/>
      </feMerge>
    </filter>

    <!-- Red glow on the live dot -->
    <filter id="dot-glow" x="-120%" y="-120%" width="340%" height="340%">
      <feGaussianBlur stdDeviation="7" result="b"/>
      <feFlood flood-color="#ff4d4d" flood-opacity=".7" result="c"/>
      <feComposite in="c" in2="b" operator="in" result="cb"/>
      <feMerge>
        <feMergeNode in="cb"/>
        <feMergeNode in="SourceGraphic"/>
      </feMerge>
    </filter>

  </defs>

  <!-- Background -->
  <rect width="512" height="512" rx="80" fill="url(#bg)"/>
  <rect width="512" height="512" rx="80" fill="url(#dots)"/>

  <!-- Code glyph </> with cyan glow -->
  <g filter="url(#glow)" stroke-linecap="round" stroke-linejoin="round" fill="none">
    <polyline points="146,176 86,256 146,336"    stroke="#00c8ff" stroke-width="44"/>
    <line     x1="310" y1="160" x2="202" y2="352" stroke="#00e5c8" stroke-width="44"/>
    <polyline points="366,176 426,256 366,336"    stroke="#00c8ff" stroke-width="44"/>
  </g>

  <!-- Live indicator dot -->
  <circle cx="395" cy="100" r="26" fill="#ff4d4d" filter="url(#dot-glow)"/>

</svg>
